VULNERABILIDADES Y AMENAZAS A LA SEGURIDAD

porque son necesarias las medidas basicas de seguridad en los dispositivos de red.

TIPOS DE AMENAZAS.
Los de robo de informacion, perdida y manipulacion de datos. Robos de identidad e interrupcion de servicio.
TIPOS DE VULNERABILIDADES
por lo general, los dispositivos de red que sufren ataques son los temrinales, servidores y computadoras de escritorio. Existen tres vulnerabilidades=debilidades: debilidades en el protocolo TPC/IP, en el OS, y debilidades del equipo de red. las de configuracion pueden incluir cuentas de usuario no seguras, contraseñas faciles, etc.
TIPOS DE MALWARE
codigo malicioso, diseñado para dañar, interrumpir, robar etc, existen los virus, que insertan copias de si mismo, formando parte de otro programa, se propaga de pc a pc. Los gusanos, a diferencia de los virus que requieren la propagacion de un archivo host infectado, los gusanos son independientes y no requieren de un programa host para propagarse. LOos trojan horses, un software que parece legitimo, se reproducen al infectar otros archivos. deben extenderse a traves de la interaccion del usuario. como abrir un archivo de correo, descargar o ejecutar archivos de la red.

SEGURIDAD DE CONTRASEÑA ADICIONAL

evitar ataques de fuerza bruta.

Hay varios pasos que se pueden tomar para ayudar a garantizar que las contraseñas permanezcan secretas en un enrutador y conmutador Cisco, incluidas estas:
Cifre todas las contraseñas de texto sin formato con el comando service password-encryption.
Establezca una longitud mínima de contraseña aceptable con el comando security passwords min-length.
Detente los ataques de adivinación de contraseñas de fuerza bruta con el comando block-for # attempts # within # #.
Deshabilite un acceso en modo EXEC privilegiado inactivo después de un período de tiempo especificado con el comando exec-timeout

PROTOCOLOS DE CORREO ELECTRONICO
metodo de guardado y desvio para enviar, guardar y recuperar mensajes electronicos a tra ves de una red. Se guardan en bases de datos del servidor. Los cliente sse comunican con el servidor para enviar y recibir correos.
Los protocolos utilizados:
*PROTOCOLO SIMPLE DE TRANSFERENCIA DE CORREO (SMTP) para enviar correo
*PROTOCOLO DE OFICINA DE CORREOS (POP) e IMAP:
para que los clientes reciban correo.
SMTP, POP e IMAP
*al enviar correo el proceso SMTP del cliente se conecta a otro proceso SMTP del servidor en el famoso puerto 25.
*con la conexion establecida, el cliente intenta enviar el correo por medio de este
*el servidor al recibir el mensaje, lo pone en una cuenta local o lo reenvia a otro servidor de correo
*si no esta en linea o esta ocupado, entonces, SMTP pone en cola los mensajes.
POP es usado para recuperar correos de un servidor, si el correo se descarga automaticamente se elimina.
*empieza escuchando pasivamente en el puerto TCP 110 las solicitudes
*el cliente decide usar el servicio, enviando una solicitud para establecer conexion TCP con el servidor
*el servidor POP envia un saludo
*el cliente y servidor intercambian comandos hasta que la conexion se cancela.
IMAP describe un metodo para recuperar correos.
*cuando el usuario se conecta a este, se descargan copias del mensaje, los originales se mantienen hasta que se eliminan manualmente
*si decide eliminar el mensaje, el servidor sincroniza la accion y lo elimina del servidor
ATAQUES D RECONOCIMIENTO
estos se clasifican en tres categorias, los ataques de reconocimiento. es el descubrimiento y el mapeo de sistemas, servicios y vulnerabilidades. Los ataques de acceso, es la manipulacion no autorizada de datos, acceso al sistema o privilegios de usuario. Denegacion de servicio, es la desactivacion o corrupcion de redes, sistemas o servicios. Para los de reconocimiento usan herramientas como NSLOOKUP y WHOIS para determinar las direcciones IP. Luego puede hacer Ping a las direcciones IP disponibles publicamente para identificar las direcciones activas.
ATAQUES DE ACCESO
estos explotan las vulnerabilidades conocidas, en los servicios de autenticacion, FTP y servicios web para tener acceso a las cuentas de web. Estos se clasifican en cuatro categorias. ATAQUES DE CONTRASEÑA: usan fuerza bruta, troyanos y rastreadores de paquetes. EXPLOTACION DE CONFIANZA: usa privilegios no autorizados para obtener acceso al sistema. REDIRECCIONAMIENTO DE PUERTOS. usa un sistema comprometido como base de datos para ataques contra otros objetivos. Man-in-the-middle: se coloca entre dos entidades legitimas para leer o modificar los datos que pasan entre las dos partes.
para proteger tus datos se recomienda usar: VPN, ASA firewall, IPS, ESA/WSA, AAA Server.

SERVIDOR DE DIRECCIONAMIENTO IP

mecanismo DNS y DHCP

SERVIDOR DE NOMBRES DE DOMINIO
*se crearon para convertir direcciones numericas en nombres sencillos.
*los FQDN como google.com son mas sencillos.
*el DNS define un servicio automatizado que coincide con nombres de recursos que tienen la direccion de red solicitada.
FORMATO DEL MENSAJE DNS
almacena diferentes tipos de registro para resolver nombres. contiene el nombre, la direccion y el tipo de regsitro.
*A: direccion IPv4 de terminal.
*NS: servidor de nombre autoritativo
*AAAA: direccion IPv6 de terminal
MX: registro de intercambio de correo
ante una solicitud, el DSN observa primero sus propios registros para resolver el nombre. Si no se puede contacta a otros servidores. esta se la devuelve al solicitante, luego se almacena temporalmente por si vuelve a soliticar.
FORMATO DE MENSAJE DNS:
se usa para todo tipo de solicitudes, para errores, transferencias de recursos etc...
JERARQUIA DNS
usa un sistema jerarquico para crear bases de datos para los nombres solicitados.
cada servidor mantiene un archivo de base de datos, y solo es responsable de administrar las asignaciones de nombre a ip para esa pqeueña porcion.
cuando recibe una solicitud, y no hay, lo reenvia a otro servidor DNS dentro de la zona. Algunos ejemplo de dominios de nivel superior:
.com (empresa o industria)
.org (una organizacion sin fines de lucro)
.au(australia)
COMANDO NSLOOKUP
una utilidad de tu OS que permite al usuario consultar manualmente los servidores DNS configurados en el dispositivo para resolver el nombre.
se usa para solucionar problemas de resolucion de nombres y verifica rel estado actual de los servidores.
PROTOCOLO DE CONFIGURACION DINAMICA DE HOST
DHCP del servidor IPv4 automatiza las asignaciones, mascaras de subred, gateways y otros parametros.
es dinamico a diferencia del estatico que se introduce manualmente la informacion.
FUNCIONAMIENTO DE DHCP
el dispositivo con DHCP se conecta a la red, el cliente transmite un mensaje de deteccion DHCP DISCOVER, para indentificar un servidor DHCP.
el servidor responde con un mensaje de oferta, DHCPOFFER, debe elegir entre ellos y enviar un mensaje de solicitud, DHCPREQUEST, que identifique el servidor explicito, luego, el servidor devuelve un mensaje de confirmacion, DHCPPACK, que reconoce el cliente que se a finalizado la concesion. Si el oferton no es valida, el servidor responde negativo con, DHCPNAK, y el proceso empieza otra vez. Recuerda los cuatro mensajes entre el cliente-servidor:
DHCPDISCOVER->DHCPOFFER->DHCPREQUEST->DHCPACK

SERVICIOS DE INTERCAMBIO DE ARCHIVOS

mecanismo de los protocolo de transferencia de archivos

PROTOCOLO FTP:
se desarrolló para permitir la transferencia de datos entre el cliente-servidor.
PASO 1:
se establece una conexion para controlar el trafico en el puerto TCP 21. el trafico consiste enc omandos de cliente y respuesta.
PASO 2:
establece una segunda conexion al servidor para la transferencia de datos real usando el puerto TCP 20. se crea cada vez que hay datos para transferir.
PASO 3:
se descarga datos o subir datos al servidor.
BLOQUEO DE MENSAJES DEL SERVIDOR
el SMB, Server mesaage block, es un protoclo cliente-servidor para compartir archivos.
tres funciones de los mensajes SMB:
*iniciar, autenticar y terminar sesiones.
*controlar el acceso a los archivos y a las impresoras.
*autorizar una aplicacion para enviar o recibir mensajes para o de otro dispositivo.
